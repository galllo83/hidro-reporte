Descripción del Problema 
En la zona metropolitana de Pachuca y municipios aledaños (como Zempoala), el 
tandeo es irregular. A veces CAASIM abre la válvula, pero por falta de presión no 
llega a casas altas. Actualmente, la única forma de saber si hay agua es abrir la 
llave o preguntar en grupos de WhatsApp vecinales. El sistema busca automatizar 
ese ‘aviso vecinal. 
La operación de las válvulas que controlan este flujo es mayoritariamente manual y 
carece de instrumentación telemétrica (sensores IoT) debido a su alto costo de 
implementación. 
Afectados  
Esta situación afecta directamente a los  ciudadanos de colonias vulnerables, 
quienes viven en incertidumbre sobre los horarios reales de servicio, y a 
los organismos operadores  (como CAASIM o comités locales), que carecen de 
visibilidad en tiempo real sobr e si las válvulas fueron abiertas correctamente o si 
existen fallos operativos (error humano). 
Causas Principales: 
1. Infraestructura obsoleta: Redes hidráulicas antiguas sin capacidad de 
monitoreo remoto. 
2. Opacidad en la información: Falta de comunicación efectiva entre el 
operador y el usuario final. 
3. Dependencia del factor humano: Errores en la apertura/cierre de válvulas 
manuales que no son detectados hasta que la población presenta quejas 
masivas. 
Justificación 
Abordar este problema mediante infraestructura física (sensores) es muy costoso 
para el presupuesto actual . Sin embargo, una solución de software basada 
en crowdsourcing ( colaboración masiva)  permite utilizar a los ciudadanos como 
"sensores humanos". Al recolectar y procesar reportes simples desde dispositivos 
móviles, se puede generar información de alto valor (inferencia de estados) con un 
costo de implementación marginal. 
Alcance General 
El sistema abarcará una aplicación móvil (PWA) para la captura de reportes 
ciudadanos en una zona piloto y un tablero de control web que procesará estos 
datos para visualizar el estado lógico de las válvulas en tiempo real. 
Objetivos del Proyecto 
Objetivo General: Desarrollar un sistema de monitoreo indirecto para la gestión 
hídrica que, mediante la recolección de reportes ciudadanos geolocalizados 
(Crowdsourcing) y un algoritmo de inferencia, permita visualizar en tiempo real el 
estado operativo de las válvulas de distribución de agua en zonas de tandeo. 
Objetivos Específicos: 
1. Implementar una Aplicación Web Progresiva (PWA)  que funcione como 
herramienta de captación de datos, permitiendo a los ciudadanos reportar 
suministro y fugas de forma georreferenciada. 
2. Desarrollar un motor de inferencia lógica en el backend que correlacione 
múltiples reportes en una misma zona geográfica para determinar el estado 
(Abierto/Cerrado) de las válvulas sin necesidad de sensores físicos. 
3. Construir un Tablero de Control (Dashboard)  para la visualización de 
datos espaciales, que facilite la toma de decisiones operativas y la detección 
de anomalías en el servicio. 
 
Relación del problema con el desarrollo sustentable 
Para fines de este proyecto, entendemos el desarrollo sustentable no solo como el 
cuidado ecológico, sino como el equilibrio donde la tecnología permite gestionar el 
agua (ambiental) de forma justa para los vecinos (social) sin que represente un 
gasto impagable para el municipio (económico). 
Dimensiones Involucradas 
Social (equidad) El sistema busca reducir la brecha de información, permitiendo 
que los vecinos tengan datos reales para documentar la calidad y frecuencia de su 
servicio, promoviendo un acceso más justo al recurso. 
Ambiental ( eficiencia): Al detectar fugas y patrones de desperdicio de manera 
temprana, se contribuye directamente a la preservación del recurso hídrico. 
Económica (optimización): Permite a los organismos operadores reducir costos 
operativos al dirigir cuadrillas de reparación de forma eficiente basada en datos 
geolocalizados. 
 
 
Relación con los objetivos de desarrollo sostenible (ODS) 
El proyecto se alinea directamente con el  ODS 6: Agua limpia y saneamiento, 
específicamente con la  meta 6.4 (uso eficiente de recursos hídricos)  y la meta 6.b 
(participación de las comunidades en la gestión del agua). 
Impacto Esperado: 
Corto Plazo: Transparencia inmediata sobre el estado del servicio en la zona 
piloto. 
Largo Plazo: Generación de una base de datos histórica que permita auditar la 
eficiencia del tandeo y justificar inversiones en infraestructura física donde más 
se necesite. 
Justificación de la Contribución a la Sustentabilidad 
El software "Hidro-Reporte" contribuye directamente a la sustentabilidad  ambiental 
y social al optimizar la gestión del recurso hídrico mediante la detección temprana 
de anomalías. Al geolocalizar fugas y monitorear el suministro real mediante 
inteligencia colectiva, el sistema reduce los tiempos de respuesta ante el 
desperdicio de agua ( eficiencia hídrica) y democratiza el acceso a la información, 
permitiendo que las comunidades vulnerables gestionen su consumo con equidad 
y transparencia, alineándose con las metas del ODS 6. 
Desde la perspectiva económica, la solución promueve la sustentabilidad al sustituir 
la inversión en costosa infraestructura de sensores físicos por un modelo de 
"sensores ciudadanos" (Crowdsourcing). Esto permite a los organismos operadores 
auditar la red y optimizar la logística de sus cuadrillas sin generar deuda técnica ni 
residuos electrónicos, logrando una gestión inteligente del agua viable, escalable y 
de bajo impacto ecológico. 
 
Propuesta de solución de software 
Se propone el desarrollo del sistema  "Hidro-Reporte", una plataforma de 
monitoreo indirecto. El software utiliza un algoritmo de inferencia que correlaciona 
múltiples reportes ciudadanos en tiempo y espacio para determinar virtualmente si 
una válvula está abierta o cerrada, sin tocar la infraestructura física. 
Tipo de Sistema:  Sistema Web Progresivo (PWA) con arquitectura Cliente -
Servidor y capacidades Geoespaciales. 
 
Actores Principales: 
1. Ciudadano: Usuario recolector de datos. 
2. Administrador (Operador): Usuario tomador de decisiones. 
3. Sistema (Backend): Actor lógico que procesa la información. 
Beneficios 
Ciudadano: Certeza sobre el servicio y un canal directo para reportar incidencias. 
Administrador: Un "Gemelo Digital" de bajo costo que le permite visualizar la 
operación de la red en un mapa en tiempo real.

Para la clasificación de las historias  de usuario se utilizó el método MoSCoW 
simplificado en tres niveles   (alta, media, baja), basándose en la dependencia 
técnica y el valor aportado al problema central: 
Prioridad Alta (Must Have):  Incluye la infraestructura base (BD Espacial), la 
seguridad de acceso y el flujo principal de "Captura de datos -> Inferencia -> 
Visualización". Sin estas historias, el sistema no cumple su propósito fundamental 
de monitoreo. 
Prioridad Media (Should Have):  Incluye validaciones de calidad de datos 
(Geofencing) y elementos de sustentabilidad (Cálculo de duración del tandeo). Son 
vitales para la fidelidad del sistema pero pueden posponerse ligeramente tras el 
MVP. 
Prioridad Baja (Could Have):  Incluye reportes históricos y visualizaciones 
alternativas. Aportan valor analítico a largo plazo, pero no impiden el funcionamiento 
operativo del software en una primera fase. 
 
Conclusiones 
El análisis de requisitos fue clave para aterrizar el problema. Pasamos de una idea 
vaga (“falta de agua”) a funciones programables. Las historias de usuario nos 
permitieron entender qué dato necesitamos realmente pedirle al ciudadano.  
El software tiene un rol protagónico en el desarrollo sustentable. Al digitalizar la 
gestión hídrica mediante inteligencia colectiva, no solo se optimiza un proceso 
técnico, sino que se democratiza la información, empoderando a la sociedad para 
participar activamente en la gestión responsable de sus recursos naturales, 
cumpliendo así con la visión de los PRONACES.
